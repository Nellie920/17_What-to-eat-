SDLC 開發時程與系統流程圖

遵循課堂統一軟體工程規範，從規劃、設計到整合測試的完整時間軸。

專案時間軸

階段一：規劃與需求分析 (Planning & PRD)

5/12 完成

• 確定開發主題為《愛與選擇之書 Whispers of Fate》戀愛視覺小說平台。

• 分析核心痛點（缺乏雙向互動、結局單一等）並撰寫產品需求文件 (PRD.md)。

• 明確劃分 6 大功能編號與小組職責分工。

階段二：系統架構與資料庫設計 (Design)

5/19 完成

• 完成 MVC 系統架構規劃 (ARCHITECTURE.md)，選用 Python Flask + SQLite。

• 設計 SQLite 資料表 Schema（使用者、存檔、成就及解鎖紀錄）。

• 規劃使用者操作流程圖 (FLOWCHART.md) 與 API 路由規劃表。

階段三：各功能獨立開發與整合 (Implementation)

5/26 進行中/已完成

• 獨立完成 F-01 到 F-06 的程式碼撰寫與本機測試。

• 進行系統大整合，合併至 Flask 骨架中並註冊藍圖。

• 實作本專案成果與狀態儀表板，全面呈現完整狀態。

階段四：功能整合測試與 Bug 修正 (Testing)

6/09、6/16 預計

• 撰寫單元測試與端到端模擬登入、遊戲存檔與解鎖成就測試。

• 優化前端多媒體音量控制與 CSS 轉場動畫的防呆處理。

階段五：專題 Demo 與部署 (Demo)

6/23 預計

• 系統雲端部署與最後測試，製作期末演示投影片與成果發佈。

資料庫結構設計 (SQLite Schema)

為實現會員登入、多槽位故事存檔以及解鎖成就功能所設計的資料庫實體關聯表格。

資料表結構圖鑑

users (會員帳號)

表1
id INTEGER PK AUTO
username TEXT UNIQUE
password_hash TEXT (密碼雜湊)
display_name TEXT (暱稱)
theme TEXT DEFAULT 'light'
created_at DATETIME

saves (玩家存檔)

表2
id INTEGER PK AUTO
user_id INTEGER FK
node_id TEXT (故事節點)
chapter INTEGER DEFAULT 1
saved_state TEXT (JSON 字串)
saved_at DATETIME

achievements (成就定義)

表3
id INTEGER PK AUTO
title TEXT UNIQUE
description TEXT (成就描述)
badge_url TEXT (勳章路徑)
points INTEGER DEFAULT 10

user_achievements

表4
id INTEGER PK AUTO
user_id INTEGER FK
achievement_id INTEGER FK
unlocked_at DATETIME
