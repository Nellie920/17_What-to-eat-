## 一、專題概述（Planning）

### 1.1 動機與背景

現在大家在手機上看故事或小說，大多只能被動接收內容，產生審美疲勞，年輕受眾不再滿足於旁觀者的角色，而是渴望在故事中尋求自我投射。而我們發現年輕一代更喜歡「能互動」的體驗，所以想開發一個讓使用者能主導劇情的平台。目前的痛點在於：讀者常對既定結局感到遺憾，卻無力改變。本專案旨在建構一個由讀者驅動的互動平台，結合生成式 AI 技術，讓每一項抉擇都能即時觸發獨特的劇情分支。我們不只是提供故事，而是提供一個讓使用者能與角色共創、體驗多重可能性，將閱讀體驗從單向接收轉化為雙向的情感互動。

### 1.2 目標使用者

對故事品質有要求、喜愛分享並希望在虛擬世界中尋求互動的年輕人

### 1.3 專題目標

目標一（**增加故事深度）：**擴展故事線路，提供多元的分支結局。

目標二（**提升參與感）：**透過客製化角色或互動選擇，強化讀者的情緒連結。

目標三（**完善進度管理）：**建立會員登入系統，確保讀者能跨裝置同步閱讀進度。

目標四（**靈活存取）：**提供「任意存檔點」功能，讓讀者能隨時回溯或保存特定劇情點。

目標五（**沉浸式體驗）：**提供符合劇情的背景音樂及精美的動畫。

## 二、需求分析（Requirements Analysis）

### 2.1 功能性需求（Functional Requirements）

| 功能編號 | 功能名稱 | 功能描述 |
| :---- | :---- | :---- |
| F-01 | 用戶登入與註冊 | 提供帳號密碼登入，用以記錄與同步個人故事進度。 |
| F-02 | 多分支互動故事系統 | 使用者可在關鍵劇情做出選擇，導向不同的故事擴展方向。 |
| F-03 | 任意存檔/讀檔功能 | 使用者可於故事任意處點擊存檔，並在存檔清單中選擇點位回溯。 |
| F-04 | 個人化設置 | 允許使用者自訂角色名稱或介面配色，提升客製化感受。 |
| F-05 | 成就解鎖系統 | 根據使用者達成的特定結局或隱藏選項，給予成就獎勵，增加重複遊玩的動力。 |
| F-06 | 背景音樂與音效切換 | 使用者可根據喜好調整或開啟故事背景音樂，提升閱讀時的沉浸感與互動體驗。 |

### 2.2 非功能性需求（Non-functional Requirements）

| 類別 | 需求描述 |
| :---- | :---- |
| 效能 | 故事文字與轉場圖片載入時間應小於 2 秒。 |
| 安全性 | 使用者密碼應進行加密處理，確保登入資訊安全。 |
| 易用性 | 介面應直覺簡單，確保使用者能快速上手。 |
| 觀賞性 | 在不同劇情中可選擇適合的音樂、精美的動畫。 |

### 2.3 範圍限制（Out of Scope）

本次不包含：高成本配音。  
本次不包含：高成本3D建模。
